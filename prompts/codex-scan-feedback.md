## Task

Use `AnimatedContainer` on the border color. Do not show the spinner during this flash — spinner is only for `processingQr == true`.

It must always be visible regardless of whether the bottom panel is collapsed or expanded. Use a `Positioned` widget inside the `Stack`. Hide it (opacity 0) only when `items.isEmpty`.

---

## Change 3 — Slide+fade animation for new items in the list

In `_BottomPanel`, replace `ListView.builder` with `AnimatedList` (or wrap each `_ScannedItemRow` with `AnimatedSwitcher`). When a new item is added (newest first at index 0), it should slide in from the top with a fade. Use `SlideTransition` + `FadeTransition` with 300ms duration.

## CRITICAL — Dart/Flutter rules (violations will break the CI build)

### Rule 1: Never use block-body factories in `@freezed` classes

If you need to modify a model with `@freezed`, the `fromJson` factory MUST use expression body `=>`, never block body `{}`. Use the spread operator for normalization:

```dart
// CORRECT
factory MyModel.fromJson(Map<String, dynamic> json) =>
    _$MyModelFromJson({
      ...json,
      if (json['id'] == null && json['_id'] != null)
        'id': json['_id'].toString(),
    });

// WRONG — breaks build_runner, do not use
factory MyModel.fromJson(Map<String, dynamic> json) {
  final normalized = Map<String, dynamic>.from(json);
  normalized['id'] = normalized['_id'].toString();
  return _$MyModelFromJson(normalized);
}
```

### Rule 2: Never call a method without defining it

If you reference a method (e.g. `_showSomething(context, id)`), that method must be defined in the same class in the same file. Check every method call you write.

### Rule 3: Never create `.g.dart` or `.freezed.dart` files manually

These are generated by `build_runner` in CI. Only modify the source `.dart` file with the annotations. Never create or edit `*.g.dart` or `*.freezed.dart` files.

### Rule 4: No new `@freezed` models needed

This task only requires changes to presentation widgets. Do not create new models.
